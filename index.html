Weekend Project: webcam-random-numbers

This small project is the synthesis of several bits and pieces I found on the web.

Inspiration

This project was mostly inspired by a YouTube video, "The Alpha Webcam Scintillator".

Hardware

The hardware consists of two main parts. First is the gutted webcam sensor with it's filters removed. You can see how to remove said filters in the YouTube video above.

The second part is a sample of Americium pulled from a cheap smoke detector. This radioactive metal is the source of the alpha particles we are detecting.

These two components are shielded with aluminum foil, then pressed together with electrical tape. I packaged it all up in a hobby project box for a nicer presentation and closer to complete darkness.

Webcam Damage

When I first created this, I didn't take into account the fact that Alpha radiation is, well, destructive. My first sensor was dying after less than 24 hours of creation, both with bad pixels and bad encoding. I started getting a lot of bad and dropped frames.

A frame grab from soon after I created the sensor.

A frame grab from roughly 16 hours after I created the sensor.

A frame grab from right before I decomissioned that sensor.

With my second sensor I took some precautions against this damage. The first was aluminum foil shielding on the circuitry. The second was a tissue paper shield between the Americium and the sensor to reduce the number and power of particles getting through. Time will tell how much of a difference this makes, but visually it appears to have worked.

Software

My software uses OpenCV to access the camera sensor and iterate over the pixels. The psuedo code for the process is: 

for pixel in frame:
  if pixel white:
    shift least significant bit of pixel index ( row * width + column ) onto buffer
    if buffer full:
      write buffer to file
    skip a few rows ( to avoid re-registering the same alpha scintillation )

I know very little about random numbers, so my methodology may be flawed.

Results

I don't currently have any big samples to test since my sensor died. I'm working on capturing a large file with my new sensor.
